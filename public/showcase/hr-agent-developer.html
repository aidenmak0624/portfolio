System Architecture ‚Äî Layered Design

5-layer architecture with clean separation of concerns

5-layer architecture: Clients ‚Üí Middleware (security) ‚Üí AI Core (agents + RAG) ‚Üí Data (DB + HRIS) ‚Üí DevOps

Agent Internal Loop ‚Äî LangGraph State Machine

Each specialist follows a Plan ‚Üí Execute ‚Üí Reflect loop with confidence-based termination

Agents don't just answer ‚Äî they reason, self-evaluate, and retry if confidence is too low. This prevents hallucinations.

Key Design Decisions

LangGraph over LangChain Agents

Chose LangGraph for deterministic state machines with explicit control flow. Each agent is a compiled graph with typed state, not a loose ReAct loop. This gives us predictable routing, retry logic, and audit-friendly decision traces.

RAG with Policy Grounding

ChromaDB vector store with sentence-transformers embeddings. Every agent response cites source documents, preventing hallucination on policy questions. Chunk overlap ensures context continuity.

Multi-Model Failover

Primary: GPT-4o-mini for speed. Premium: GPT-4o for complex queries. Fallback: Google Gemini if OpenAI is unavailable. All routing is automatic with cost tracking per request.

Interface-Based Connectors

HRIS integrations implement a common HRISInterface ABC. Adding a new system (SAP, ADP, etc.) means implementing one interface ‚Äî zero changes to agent logic.
